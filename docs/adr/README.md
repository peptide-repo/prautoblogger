# PRAutoBlogger — App-Local ADR Log

This folder is the per-repo decision log for the **PRAutoBlogger** plugin.
It records architectural choices that affect this codebase alone.

**Cross-cutting decisions** (spanning multiple plugins, shared infra, or
CI/deploy standards) live centrally in
`Peptide Repo CTO/docs/engineering/decisions/`.

For the routing rule, required sections, and status lifecycle, read the
canonical process document:
`Peptide Repo CTO/docs/engineering/ADR-PROCESS.md`

The canonical template for new ADRs is:
`Peptide Repo CTO/docs/engineering/adr-template.md`

## Numbering

ADRs in this folder use a per-repo zero-padded sequence starting at `0001`.
A new ADR always gets the next unused number; numbers are never reused.
